## Goal
Build the Operations Manager crew — 3 agents (PreFlightAgent, CronWatchdog, Reporter) with deterministic tools for 7 infrastructure health checks. Write 17 tests first, get user review, then implement the crew. All 17 tests must pass engine-only (no LLM calls).

## Purpose
OM is the first proper multi-agent crew. It runs at 8:00 AM IST as a separate cron, independently checking system readiness before Phase 1's 9:30 AM entry. Without OM, there's no pre-flight verification that the trading infrastructure is healthy.

## Output
- `tools/om_tools.py` — 7 deterministic @tool functions (token_refresh_status, verify_code_hash, data_capture_health, disk_usage_check, network_connectivity_check, cron_health_check, aggregate_health_report)
- `crews/om_crew.py` — 3-agent CrewAI crew (PreFlightAgent, CronWatchdog, Reporter) using Process.hierarchical
- `tests/test_om_crew.py` — 17 passing tests (env-var mocking, engine-only)
</objective>

    Create tools/om_tools.py with 7 deterministic tools:
    
    1. token_refresh_status() → {"shoonya": bool, "flattrade": bool, "ok": bool, "evidence": str}
       Check token files exist (cred.yml, tokens.json) and have recent timestamps
       
    2. verify_code_hash() → {"unchanged": bool, "ok": bool, "evidence": str}
       Hash all .py files in antariskh/ against last-known hash. ANTARIKSH_MOCK_CODE_CHANGED=1 forces fail
       
    3. data_capture_health() → {"running": bool, "ok": bool, "evidence": str}
       Check DuckDB data file has recent writes. ANTARIKSH_MOCK_DATA_STOPPED=1 forces fail
       
    4. disk_usage_check() → {"pct_used": float, "free_gb": float, "ok": bool, "evidence": str}
       os.statvfs on logs/ directory. ANTARIKSH_MOCK_DISK_FULL=1 forces pct_used=98.0, ok=False
       
    5. network_connectivity_check() → {"broker": {"shoonya": int, "flattrade": int}, "ok": bool, "evidence": str}
       Ping broker API endpoints (or mock with ANTARIKSH_MOCK_BROKER_DOWN=1)
       
    6. cron_health_check() → {"active": list, "missing": list, "ok": bool, "evidence": str}
       Parse crontab, compare against expected entries in antariksh_rules.yaml
       ANTARIKSH_MOCK_CRON_DEAD=1 forces all missing
       
    7. aggregate_health_report(results: List[Dict]) → {"overall": "GO"|"NOGO", "checks": [], "telegram_md": str}
       Classify: CRITICAL = broker/tokens/disk fail → NOGO. WARNING = data/code → GO with note.
       Generate Telegram-ready markdown with timestamps and evidence values.
    
    All tools check os.environ for ANTARIKSH_MOCK_* vars before real system calls.
    Every tool returns an evidence string with actual values or mock values.
    
    Avoid: LLM calls, importing crew_structure.py, modifying existing files.
  </action>
