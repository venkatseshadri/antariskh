## ARCHITECTURE DECISIONS

### 1. Entry Gate: 0 LLM at Runtime
All 9 family tools (`query_trend`, `query_momentum`, `query_volatility`, `query_volume`, `query_options`, `query_flow`, `query_macro`, `query_traffic_light`, `query_all_families`) read **Redis only**. No DuckDB. No LLM. ~100ms response time. LLM reserved for **post-session RL weight learning** only.

### 2. Redis as Live Data Source
Live trading reads from Redis (`v3_ohlcv_queue`). DuckDB used only for post-market analysis and pattern enrichment. This avoids lock contention during live trading hours.

### 3. Paper Mode: Take ALL Trades
No signal filtering in paper mode. Take all non-conflicting trades to maximize data collection. Only skip when BULLISH and BEARISH both fire (signal conflict).

### 4. Position Manager: Single SL/TP Owner
Position Manager (not risk system) owns all stop-loss and take-profit decisions. 7-priority checks:
- **P1:** Theta decay roll (37.5% premium erosion)
- **P2:** Hedge tighten (within 150pt gap)
- **P3:** Signal morph (BULLISH↔NEUTRAL↔BEARISH)
- **P4:** Stop Loss (50% above entry)
- **P5:** Take Profit (50% below entry)
- **P6:** Cumulative floor (account-level drawdown)
- **P7:** Market close (15:30 flat)

### 5. Signal-Driven Strategy Selection
- BULLISH + confidence ≥ min → PUT_CREDIT_SPREAD
- BEARISH + confidence ≥ min → CALL_CREDIT_SPREAD
- NEUTRAL → IRON_BUTTERFLY
- CONFLICT (both fire) → SKIP
